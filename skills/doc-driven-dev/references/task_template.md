# 任务：用一句话写清要交付什么

## 1. 任务定义

- 目标：一句话写清这次要完成什么。
- 验收：写清怎样算完成。
- 边界：写清不做什么、不能动什么；不适用写 `N/A`。

## 2. 执行方案

- 方案：写清准备怎么做。
- 验证：写清准备怎么验证。
- 回写：若预判会影响 `doc/project.md`，写候选章节；不适用写 `N/A`。

## 3. 执行结果

- 实施：写清实际做了什么。
- 验证：写清实际验证了什么，结果如何。
- 结论：写清当前状态、剩余问题与下一步；不适用写 `N/A`。

## 4. 任务终结

- 交付：汇总最终产出。
- 验收：说明是否完成，以及对应证据。
- 遗留：记录遗留问题或后续建议；不适用写 `N/A`。
