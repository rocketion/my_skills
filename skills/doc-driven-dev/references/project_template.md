# 工作域基线

> 这是“当前工作域的稳定基线”，不是整个仓库的全量百科。
> 第 1 章只写短、硬、稳定的全局规则。
> 只保留与当前工作相关的章节，不相关内容直接删除，不要为了完整性堆 `N/A`。

## 1. 全局规则

- 交互与编码：例如语言、编码、格式与命名约定。
- 工作方式：例如文档先行、`task` 是最小实施单元、`finding` 为可选并行记录。
- 修改边界：例如只读源、可修改区、禁止触碰区。
- 回写规则：例如哪些变化回写 `project.md`、哪些变化回写 `task` 或 `findings`。
- 提交规则：例如提交描述来源、标识要求与阻断条件。
- 文档指针：只列需要优先看的核心文档，例如 `doc/tasks.md`、`doc/task-编号-概述.md`、`doc/findings.md`。

## 2. 工作域定义

- 类型：新项目 / 重构 / 迁移 / 文档 / 其他。
- 背景：当前为什么做这件事，以及已确认的上下文事实。
- 目标：当前阶段希望稳定达成的结果。
- 非目标：明确不做什么，避免 scope 漂移。

## 3. 范围与边界

- 覆盖对象：仓库、模块、文档域、流程、系统或交付物。
- 非覆盖对象：当前工作域明确不纳入的范围。
- 权威来源：需求、现有代码、上游文档、用户说明、外部规范等。
- 操作边界：只读源、可修改区、禁止触碰区。

## 4. 对象地图与稳定事实

- 对象地图：当前工作域相关的模块、文档、流程、系统、角色或产物。
- 稳定事实：跨 task 仍然成立的职责、接口、术语、格式、依赖、结构、发布或协作口径。
- 已知限制：历史包袱、暂不处理项、长期保留风险、兼容性约束或已知问题。

## 5. 协作规约

- 文档组织：哪些信息写 `project.md`，哪些写 `task` 文档，哪些写 `findings.md`。
- 回写触发：哪些变化必须回写第 1 章“全局规则”，哪些变化只需回写工作域章节。
- 验证口径：常用验证方式、证据要求与通过标准。
- 提交口径：提交信息组织方式、必须携带的标识与约束。

## 6. 条件章节（按需保留）

### 6.1 新项目

- 架构：模块边界、关键对象、依赖关系、部署或发布方式。
- 接口与数据：关键接口、数据流、配置项、外部依赖与版本策略。

### 6.2 重构 / 迁移

- 现状问题：现有结构、痛点、历史包袱与不可回避限制。
- 兼容边界：不可破坏项、兼容策略、迁移顺序与回滚点。

### 6.3 文档任务

- 目标读者：文档面向谁，以及他们的使用场景。
- 信息架构：目录结构、章节职责、交叉引用与导航方式。
- 来源材料：权威文档、事实来源、术语口径与发布要求。
